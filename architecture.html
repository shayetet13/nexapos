5) System Flows

Flow A: POS Checkout (Critical Path)

POS UI -> POST /shops/:shopId/orders
  -> Zod validate -> stock check -> calculate points/discount/VAT
  -> DB transaction (orders + order_items)
  -> deduct stock + BOM consumables
  -> update customer points/tier
  -> broadcast ORDER_CREATED/STOCK_UPDATE/CHECKOUT_PAID
  -> return order summary + receipt token

Flow B: QR Login

Desktop create QR session -> mobile scan -> session confirm (WS)
-> backend generate login token/magic link -> desktop verifyOtp -> role-based redirect

Flow C: Customer Display

POS broadcasts CHECKOUT_CASH/CHECKOUT_QR/CHECKOUT_PAID/CHECKOUT_CLOSE
-> /ws-display clients render payment state in real-time
-> reconnect receives last known state

Flow D: Refund + OTP

Admin requests OTP -> backend issues 6-digit token + expiry
-> send via Telegram -> admin confirms OTP
-> set order status refunded + create logs/notifications

Architecture Notes

  • หัวใจของระบบคือ tenant isolation: ทุก query และทุก route ต้องผูกกับ shop_id.
  • Cloudflare Worker เป็น edge gateway ช่วยเรื่อง rate limit, CORS, และ internal token pass-through.
  • Fastify backend แยกเส้นทางชัดเจน: routes → services → repositories → db schema.
  • WebSocket มี 3 ช่องทางหลัก: POS realtime, customer display, และ QR auth flow.
